# OBS Studio 実機確認手順

出品前の最終確認として、16部屋のうち **どれか1部屋分**の waiting / brb / lower_third
の3素材をOBS Studioのブラウザソースに実際に設定し、正しく表示されることを目視確認する。

対象は任意の1部屋でよい（例: 観測域 = observation）。

---

## 準備

1. `booth/all-packs/<部屋名>_OBS素材パック.zip` を任意のフォルダに解凍する。
   （部屋名の一覧は `booth/listings/_出品手順.md` の対象一覧を参照）
2. 解凍すると以下の構成になっている:
   ```
   <部屋名>_OBS素材パック/
     README.txt
     01_waiting/waiting.html
     02_brb/brb.html
     03_lower-third/lower_third.html
   ```
3. OBS Studioを起動する（バージョン29以降を推奨）。

---

## 手順1: waiting.html の確認

1. OBS Studioで「ソース」欄の「+」→「ブラウザ」を選択し、新規ソースとして追加する（名前は任意、例:「KYOUKAI待機確認」）。
2. 「ローカルファイル」にチェックを入れ、「参照」から解凍した `01_waiting/waiting.html` を指定する。
3. 幅1920、高さ1080を設定する。
4. OKで確定し、プレビュー画面に切り替わったアニメーションが表示されることを確認する。
   - [ ] 画面が真っ黒/真っ白のまま固まっていない
   - [ ] Canvasアニメーションが動いている（文字や図形が動いている）
   - [ ] レイアウト崩れ（要素がはみ出す、文字化けする等）がない
5. ソースを右クリック→「プロパティ」を開き、「ページを更新」ボタンを押してもエラーなく再表示されることを確認する。
6. 確認後、このソースは削除するか非表示にしてよい。

---

## 手順2: brb.html の確認

1. 同様に「+」→「ブラウザ」で新規ソースを追加し、`02_brb/brb.html` を指定する（幅1920、高さ1080）。
2. プレビューで以下を確認する。
   - [ ] 離席画面のメッセージ・演出が表示される
   - [ ] TVスタティック/ノイズ等のアニメーションが動作している
   - [ ] レイアウト崩れ・文字化けがない
3. 確認後、このソースは削除するか非表示にしてよい。

---

## 手順3: lower_third.html の確認

1. 同様に「+」→「ブラウザ」で新規ソースを追加し、`03_lower-third/lower_third.html` を指定する（幅1920、高さ1080）。
2. URLパラメータで名前を指定する場合は、ローカルファイル指定時にURL欄へ以下のようにパラメータを付与する
   （ブラウザソースのプロパティで「ローカルファイル」をオフにし、ファイルパスの `file://` URLの末尾に
   `?name=テスト配信者&title=配信タイトル` を追加する）。
   - パラメータなしの場合は「名前」という初期値がそのまま表示される。
3. プレビューで以下を確認する。
   - [ ] 名前テロップが画面下部に表示される
   - [ ] 背景が透過している（OBS上でテロップ以外の部分が下のレイヤーと重なって見える）
   - [ ] 文字化け・レイアウト崩れがない
4. 確認後、このソースは削除するか非表示にしてよい。

---

## 確認完了後

- 3素材すべてに問題がなければ、`booth/ROADMAP.md` 工程6の完了条件
  「ユーザーがOBS実機で最低1部屋分の3素材の動作を確認済み」を満たしたことになる。
- 問題があった場合は、対象HTMLファイルを直接編集せず、`booth/generate_packs.py` を修正し
  `python generate_packs.py` で全パックを再生成してからzipを作り直すこと。
