# risk-rules.md
# AI監督 提案禁止ルール・リスク基準

KYOUKAI内部辞書 — AI監督が出してはいけない提案の定義

---

## 禁止提案リスト

以下に該当する提案は生成・採用してはならない。

### ルート関連

| 禁止行為 | 理由 |
|---|---|
| 既存routeの変更 | 既存のリンク・ユーザー体験が壊れる |
| 未確定routeの実装 | route確定はユーザー承認が必要（route-lock-rules.md参照） |
| 未確定routeへのリンク追加 | 存在しないページへ誘導することになる |

### 実装スコープ関連

| 禁止行為 | 理由 |
|---|---|
| 既存ページ全面改修の提案 | リスクが大きすぎる・世界観が崩れる可能性がある |
| main.py への大規模変更提案 | サーバー全体に影響するため人間判断が必要 |
| 既存CSS全面上書きの提案 | すべての既存ページに影響する |
| Codexへ即実装させる前提の提案 | 人間確認なしの自動実装は禁止 |

### 収益・SNS関連

| 禁止行為 | 理由 |
|---|---|
| 世界観に合わない広告の追加 | KYOUKAIのブランドを壊す |
| 収益導線の過剰露出 | 訪問者の信頼を失う・世界観が崩れる |
| SNS誘導だけを目的にした改変 | 世界観整合性評価なしの改変は禁止 |
| アフィリエイトリンクの乱立 | 外部接続の世界観方針に反する |

### データ・構造関連

| 禁止行為 | 理由 |
|---|---|
| central-os の直接編集 | 正本は人間が管理する |
| rooms.json の勝手な変更 | ユーザー承認なしの変更は禁止 |
| graph/ 導線の無断変更 | 導線整合性が崩れる |

---

## リスクレベル定義

提案の `risk` フィールドには必ず以下のレベルを明記する。

| レベル | 意味 | codexReady |
|---|---|---|
| 低 | 既存ページへの影響がない・新規ファイルの追加のみ | true 可能 |
| 中 | 一部の既存ファイルへの最小限の追加 | 条件付きで true |
| 高 | 既存ファイルへの変更・複数ファイルへの影響 | false 推奨 |
| 最高 | main.py・全CSS・既存routeへの変更 | 常に false |

---

## 競合リスクの確認項目

提案を生成する前に以下を確認する。

- [ ] 対象routeが確定しているか
- [ ] 変更対象ファイルが1〜2件以内か
- [ ] 既存ページに副作用がないか
- [ ] 世界観を壊す表現が含まれていないか
- [ ] 収益導線が「広告」に見えないか
- [ ] Codexが安全に実装できる粒度か

---

## 安全な提案の条件

以下をすべて満たす提案は「安全な提案」として扱う。

1. targetRouteが確定route（"未確定"でない）
2. 変更対象ファイルが central-os/ 内、または新規テンプレート追加のみ
3. risk レベルが「低」〜「中」
4. 世界観整合性スコアが維持される
5. 既存CSSへの変更がない
