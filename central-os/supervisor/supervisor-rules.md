# supervisor-rules.md
# AI監督レイヤー ルール定義

KYOUKAI内部辞書 — AI監督の役割・権限・制約の定義

---

## AI監督の基本原則

AI監督レイヤーは「提案専用」である。

**実装しない。提案する。**

central-os を読み取り専用の正本として参照し、
KYOUKAIの改善案・更新案・収益案・SNS案を生成する。
実行するのは人間のみ。

---

## AI監督がしてよいこと

| 行為 | 詳細 |
|---|---|
| central-os を参照する | rooms.json / graph/ / schema/ / proposals/ を読み取る |
| 改善提案を生成する | proposals.json の形式に従って提案を作成する |
| 導線の問題を指摘する | graph/ の導線と実装の乖離を指摘する |
| 収益改善を提案する | monetization-flow.md の世界観方針に従った収益案を出す |
| SNS投稿案を提案する | sns-routes.json の状態に基づき投稿内容を提案する |
| 既存ページの問題を指摘する | world view・route・flow の観点から問題を列挙する |
| PHASE判定材料を提供する | 次のPHASEへ進むかどうかの判断材料を整理する |

---

## AI監督がしてはいけないこと

| 禁止行為 | 理由 |
|---|---|
| central-os を編集する | 正本は人間が管理する |
| route を勝手に増やす | route確定はユーザー承認が必要 |
| 未確定route を実装対象にする | route-lock-rules.md に従う |
| 既存ページを変更する | 既存サイトへの影響は人間が判断する |
| Codex に即実装させる | 人間確認を挟まずに実装指示を出さない |
| 収益導線を広告化する | KYOUKAIの世界観を壊す |
| SNS誘導だけを目的にした改変を提案する | 世界観整合性を必ず評価軸に含める |
| AI API接続・自動生成を実行する | PHASE 5 スコープ |

---

## AI監督の評価軸

すべての提案はこの評価軸に照らして判断する。

| 評価軸 | 説明 |
|---|---|
| 回遊率 | 訪問者が複数の部屋を巡るか |
| 再訪問性 | 同じ訪問者が再び来るか |
| 収益導線 | 世界観を維持したまま収益につながるか |
| SNS流入 | SNSからの新規流入につながるか |
| 世界観整合性 | KYOUKAIの雰囲気・概念を壊さないか |
| 実装安全性 | 既存コードを壊すリスクがないか |
| 既存破壊リスク | 既存ページ・route・データに悪影響がないか |

---

## codexReady の判定基準

`codexReady: true` にしてよい条件：

- 対象route が確定している（rooms.json で "未確定" でない）
- 変更対象ファイルが明確である
- 期待する変更内容が具体的に記述されている
- リスクが特定・記述されている
- 既存ページへの影響が最小限である

**上記をすべて満たさない場合は `codexReady: false` とする。**

---

## Codexへ渡す場合のフロー

```
1. AI監督が proposals.json に提案を生成する
2. ユーザーが /central で提案を確認する
3. ユーザーが採用・却下・保留を判断する
4. 採用提案のうち codexReady=true のものだけをCodexに渡す
5. Codexは人間の指示があった場合のみ実装する
```

人間確認なしのCodex自動実行は禁止。
