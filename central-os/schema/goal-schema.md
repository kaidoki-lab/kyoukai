# monthly-goal.json 項目定義

KYOUKAI内部辞書 — monthly-goal.json フィールド定義

---

## 対象ファイル

`central-os/monthly-goal.json`

---

## このファイルの目的

KYOUKAI の月間目標を記録・管理する。

目標の達成条件を明文化することで、
月末の振り返り・次月の計画立案の基準とする。

---

## フィールド定義

| フィールド | 型 | 意味 | 備考 |
|---|---|---|---|
| id | string | 目標の一意識別子 | `goal_YYYY_MM` 形式。例：`goal_2026_05` |
| title | string | 目標の名称 | `YYYY年MM月の目標` 形式を推奨。 |
| targetRooms | array | この月に優先的に作業する部屋の name 一覧 | rooms.json の name と一致させる。 |
| successCondition | array | 達成とみなす条件のリスト | 文字列の配列。曖昧な表現は避け、確認可能な状態で記述する。 |
| notes | string | 月の方針・背景・判断基準の補足 | 自由記述。なぜその目標を選んだかを記載することを推奨。 |

---

## id の命名ルール

| パターン | 意味 |
|---|---|
| `goal_2026_05` | 2026年5月の目標 |
| `goal_2026_06` | 2026年6月の目標 |

- 年4桁・月2桁のゼロ埋めで統一する
- 月が変わったら新しい item を追加する（既存を上書きしない）

---

## successCondition の記述ルール

- 「〜する」「〜が存在する」など確認可能な形式で書く
- 「頑張る」「整理する」など抽象的な表現は避ける
- 1条件 = 1文字列。複数条件は配列で列挙する

---

## 過去データの保持

達成・未達成にかかわらず、過去の月次目標は items 配列に残す。

削除しない。月末に status フィールドを追加して記録することを将来的に検討する。
