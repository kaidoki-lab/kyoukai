# 境界セントラルOS 仕様書

## JSON構造

全JSONは以下の形式で統一する。直配列は禁止。

```json
{
  "version": "1.0",
  "items": []
}
```

理由：
- 後からmetadataを追加しやすい
- version管理が容易
- 将来的なAPI接続を想定
- AI側の構造崩壊を防ぐ

---

## 状態ラベル（status）

| 値 | 意味 |
|---|---|
| 未着手 | まだ手を付けていない |
| 制作中 | コンテンツ・素材を作っている |
| 実装中 | コードを書いている |
| 修正中 | 既存を直している |
| 確認待ち | レビュー・動作確認待ち |
| 実装済み | 本番に反映済み |
| 保留 | 一時停止・判断待ち |

## priorityルール

| 値 | 使いどころ |
|---|---|
| 最優先 | 今週中に完了必須 |
| 高 | 今月中に完了したい |
| 中 | できれば今月 |

## route方針

- 実在が確認できたルートのみ route フィールドに記載する。
- 確認できていないルートは `"未確定"` とする。
- 確定ルートは main.py の PAGE_MAP・FastAPI ルート定義を根拠とする。
- ユーザーが明示的に指定した場合のみ未実装ルートを記載できる。

---

## 変更可能範囲

Claude Code が変更してよいのは `central-os/` 内のファイルのみ。

それ以外（`app/` `pages/` `src/` `static/` `templates/` `main.py` 等）は変更禁止。

---

## central-os/ の役割

KYOUKAI 本体とは独立したデータ管理レイヤー。

- 本体コードへの干渉なし
- KYOUKAI の状態・方針・導線を記録・整理する補助フォルダ
- 将来的な `/central` ページ連携を見据えた構造設計

---

## 禁止事項

- 既存 app / pages / src / public / components / styles への変更
- 既存ルート・既存CSS・既存HTML・既存画像への干渉
- v1段階でのUI・API・DB・保存機能の実装
- 作業競合
