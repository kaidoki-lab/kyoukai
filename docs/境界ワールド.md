# 境界ワールド

## フロア自動増設ルール

KYOUKAIでは1フロアの部屋数を5部屋までに固定する。
同一フロアに6部屋目は存在しない。
6部屋目が追加された場合は、自動的に次フロアの1部屋目として登録する。
例：6階に5部屋ある状態で新しい部屋を追加した場合、その部屋は7階1部屋目になる。
この処理は手動ではなく、部屋追加時に自動で行う。

## 最上階固定部屋ルール

KYOUKAIでは通常フロアとは別に、常に一番上へ配置される最上階専用部屋を持つ。
この部屋は1フロア5部屋固定ルールの対象外とする。
通常部屋がどれだけ追加され、階層が増えても、最上階専用部屋は常に通常部屋の最上階より一つ上の階に配置する。
最上階には同一フロア内の他の部屋や空きスロットを表示しない。
最上階は常にこの専用部屋1つだけで構成する。

## シナリオ Route_B「記録されていない人」

Route_B「記録されていない人」は、Route_A「混線している観測」の完了後に続く通常シナリオである。

Route_A 完了後、管理人室に滞在したまま20秒経過すると赤電話が鳴る。管理人室を一度出て再入室する必要はない。

進行順は以下とする。

```text
route_b_phone_001
↓
route_b_room_archive_001
↓
route_b_room_hyougi_001
↓
route_b_room_gokuraku_001
↓
route_b_manager_return_001
```

Route_Bで使用する部屋は以下。

```text
kanrinin
archive
hyougi
gokuraku
```

Route_B完了時は `floor_05` を開放する。ただし最上階、最終シナリオ、消滅の鍵、巨大鍵穴、最上階鍵穴は開放しない。

以下の値は Route_B 完了後も false のまま維持する。

```json
{
  "final_route_available": false,
  "top_floor_unlocked": false,
  "annihilation_key_obtained": false,
  "top_floor_keyhole_active": false
}
```

Route_Bの会話文、部屋イベント、管理人イベント、日誌はコードへ直接書かず、イベントデータとして管理する。

## シナリオ Route_C「壊れる前の形」

Route_C「壊れる前の形」は、Route_B「記録されていない人」の完了後に続く通常シナリオである。

Route_C は 5F の `null`, `ma`, `particles` を使用する。Route_A の `observation`, `signal`、Route_B の `archive`, `hyougi`, `gokuraku` は主要経路として使用しない。管理人室 `kanrinin` は共通ハブとして使用する。

開始条件は以下。

- シナリオモードである。
- Route_A が completed である。
- Route_B が completed である。
- Route_C が not_started である。
- active_route_id が null である。
- `floor_05` が開放済みである。
- Route_B 完了後、一度管理人室を退出している。
- 管理人室へ再入室後、20秒経過している。
- 他の電話イベントが発生していない。

Route_B 完了直後の同一滞在中には Route_C を開始しない。再入室後に20秒の着信判定を行う。

進行順は以下。

```text
route_c_phone_001
↓
route_c_room_null_001
↓
route_c_room_ma_001
↓
route_c_room_particles_001
↓
route_c_manager_return_001
```

Route_C 完了時は `floor_06` を開放する。6Fの次ルートは自動開始しない。

6Fに存在する部屋は以下。

- `ripple`
- `colony`
- `dot-art`
- `matsuri`
- `namahage`

Route_C完了後も、最上階、最終シナリオ、消滅の鍵、巨大鍵穴、最上階鍵穴は開放しない。

以下の値は Route_C 完了後も false のまま維持する。

```json
{
  "final_route_available": false,
  "top_floor_unlocked": false,
  "annihilation_key_obtained": false,
  "top_floor_keyhole_active": false
}
```
## シナリオ Route_D「集まっているもの」

Route_D「集まっているもの」は、Route_C「壊れる前の形」の完了後に続く通常シナリオである。

Route_D は 6F の `ripple`, `colony`, `dot-art`, `matsuri`, `namahage` を使用する。管理人室 `kanrinin` は共通ハブとして使用する。Route_AからRoute_Cまでに主要経路として使用した部屋は、Route_Dの主要導線には使わない。

開始条件は以下。

- シナリオモードである。
- Route_A、Route_B、Route_C が completed である。
- Route_D が not_started である。
- active_route_id が null である。
- `floor_06` が開放済みである。
- Route_C 完了後、一度管理人室を退出している。
- 管理人室へ再入室後、20秒経過している。
- 他の電話イベントが発生していない。

Route_C 完了直後の同一滞在中には Route_D を開始しない。再入室後に20秒の着信判定を行う。

進行順は以下。

```text
route_d_phone_001
↓
route_d_room_ripple_001
↓
route_d_room_colony_001
↓
route_d_room_dot_art_001
↓
route_d_room_matsuri_001
↓
route_d_room_namahage_001
↓
route_d_manager_return_001
```

Route_D 完了時は通常シナリオ完了数を増やす。ただし新規階層は開放しない。

Route_D完了後も、最終シナリオ、最上階、消滅の鍵、巨大鍵穴、最上階鍵穴は開放しない。

以下の値は Route_D 完了後も false のまま維持する。

```json
{
  "final_route_available": false,
  "top_floor_unlocked": false,
  "annihilation_key_obtained": false,
  "top_floor_keyhole_active": false
}
```

Route_D以降の進行は、階層開放ではなく接触条件へ切り替える。管理人会話の要点は「触ったものが、次の形になります」。

## シナリオ Route_E「観測の完了」

Route_E「観測の完了」は、Route_D「集まっているもの」の完了後に続く終幕専用ルートである。

Route_Eは通常ルートではなく、KYOUKAIシナリオの最終到達点として扱う。Route_E完了後にRoute_Fや追加ルートへ接続しない。

開始条件は以下。

- シナリオモードである。
- Route_A、Route_B、Route_C、Route_D が completed である。
- Route_E が available である。
- active_route_id が null である。
- final_route_available が true である。
- ending_completed が true ではない。
- Route_D 完了後、一度管理人室を退出している。
- 管理人室へ再入室後、20秒経過している。
- 他の電話イベントが発生していない。

Route_D 完了直後には Route_E を自動開始しない。Route_D 完了時は以下の待機状態にする。

```json
{
  "final_route_available": true,
  "active_route_id": null,
  "top_floor_unlocked": false,
  "top_floor_keyhole_active": false
}
```

Route_E は管理人室から開始する。管理人室へ再入室した時点では即時開始せず、既存ルールどおり20秒後に最後の赤電話を鳴らす。赤電話を取った時点で以下に更新する。

```json
{
  "active_route_id": "route_e",
  "route_e.status": "active"
}
```

基本進行順は以下。

```text
Route_D完了
↓
管理人室へ再入室
↓
20秒後に最後の赤電話
↓
Route_E開始
↓
最上階
↓
鍵穴
↓
消滅の鍵
↓
逆観測室
↓
管理人室帰還
↓
管理日誌
↓
Route_E完了
↓
KYOUKAI終幕完了
```

Route_Eの正式完了条件は以下。

```text
route_e_phone_completed
top_floor_event_completed
annihilation_key_used
observer_final_event_completed
manager_return_completed
final_diary_entry_unlocked
```

すべて成立した時点で以下へ更新する。

```json
{
  "route_e.status": "completed",
  "active_route_id": null,
  "ending_completed": true,
  "kyoukai_completed_at": "ISO_DATE_TIME"
}
```

電話の会話文、発信者、最上階・鍵穴の詳細、消滅の鍵の正式仕様、逆観測室の最終文、管理日誌本文、終幕後の再訪演出は後続仕様で確定する。Route_Eでは分岐、失敗エンド、次ルート、新しい収集システム、大型の追加部屋は作らない。
