### コンセプト

「日本ゲームセンター情報空間が限界突破して、最終的にバカになる観測領域」

### 背景

- 使用画像: `static/kyoukai_null_arcade_v4.png`
- STARTボタン画像: `static/kyoukai_null_start_button.png`
- ステータスパネル画像: `static/kyoukai_null_status_panel.png`
- 背景は極彩色ゲームセンター。
- PC版では背景画像の全体ができるだけ見えるよう `contain` 寄り。
- スマホ版では縦画面優先。

### STARTチャレンジ

- START連打でゲーム開始。
- カウントダウン初期値: 10秒。
- START連打では時間は延びない。
- START連打により内部の進行・表示崩れ・演出が進む。
- 10秒で失敗すると、まきぐそ / FAILED / ZANNEN 系の失敗演出。

### 延長アイコン

- 表示名: `COLLAPSE +10 SEC`
- START後、ゲーム中だけ出現。
- START連打ではなく、この崩壊アイコンを触った時だけ10秒延長。
- 最大5回まで延長可能。
- 押すたびに別の場所へ飛ぶ。
- 約1.65秒ごとにも位置が変わる。
- スマホでは画面外・下部UIに隠れにくい範囲へ制限。
- PC版では背景画像外の余白にも飛ぶ。

### BOUNDARY NODE

- PC版では中央背景画像に被せず、背景画像の外側エリアへ配置する方針。
- スマホ版では上部看板として表示。
- 管理UIではなく、ゲームセンター看板風。
- 現在の修正ではPC版の左側余白に収める指定を追加。

### ステータス画面

- 数値としての崩壊度は見せない。
- `corruption %` や `globalInfection %` はユーザーに表示しない。
- ステータス表示は黒背景カードへ整理。
- スマホでは左下寄り、STARTボタンと並べて邪魔にならないよう調整済み。

### MAX / 失敗

失敗:

- まきぐそ演出。
- `FAILED / ZANNEN`。
- ゲーセンの失敗画面風。

MAX:

- まきぐそではない。
- 黒画面に `崩壊 / COLLAPSE`。
- RETURNのみ。
- localStorageに collapsed 状態を保存。

### localStorage

キー:

`kyoukai_null_state`

値の例:

```json
{
  "globalInfection": 37.4,
  "collapseLevel": 37.4,
  "extensionCount": 3,
  "failedCount": 1,
  "maxReached": false,
  "collapsed": false,
  "lastPlayedAt": "2026-05-29T00:00:00.000Z"
}
```

リセット:

- `/null?reset_null=1`
- `kyoukai_null_state` だけを削除。
